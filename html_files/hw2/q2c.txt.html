1    The exploit we take advantage of in this section is the fact that we know 
2    what we want the original message to be - it should be 1. 
3    The RSA verification algorithm assumes an original message, m, was signed using a 
4    private key d, by performing c = m^d (mod n) 
5    So in order to verify the authenticity of the message the method performs c' = c^e (mod n) 
6     - using the public key e attribute and n the public modulo value. 
7    Than comparing the result c' to the message sent, verifying they are indeed equal. 
8    In our case we want to forge the message to be of value 1 (CODE_APPROVAL). 
9    So all we have to do is to look for a value, 'c', which holds that c^e (mod n) == 1. 
10   A trivial solution is to set c = 1. 
11    
12   To sum it up - our response includes status attribute with value 1 (CODE_APPROVAL) 
13   and signature attribute with value 1. 
14   So it holds that 1^e (mod n) = 1 mod(n) = 1 == response.status.
