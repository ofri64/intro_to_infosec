1    - Similar to previous section all we want to do is patch the return value of the program, 
2    regardless of the program logic itself 
3    - Again, looking in IDA we see that if the message is valid we move into EAX the value 0 
4    (EAX will contain the return value of the program). 
5    - If the message is not valie we move into EAX the value 1 - this is where we want to patch our code 
6    - We see the offset is 0x6DD, the first byte is the opcode 
7    (0xB8, telling us it is a mov EAX instruction), the last 4 bytes are the immediate operand in little endian 
8     (0x01 0x00 0x00 0x00 which means 1) 
9    - So the patch we need is to replace the byte 0x6DE to 0x00, 
10    resulting in the following commad: MOV EAX 0. 
11   - This doesn't change the output to the screen or the validation test, 
12   only changes the return value of the function in cases where the validation test failed.
