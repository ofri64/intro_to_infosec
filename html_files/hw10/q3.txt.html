1    - In this question we need to post an image message, 
2     but control its behaviour with a script 
3     
4    - We need to maintain state - if the user already seen the image/gif we 
5     want to make it invisible, otherwise present the message 
6     
7    - We notice another important thing - 
8    looking at the html source code we see that a variable named "username" 
9    is already defined, matching the current logged on user. 
10    This is an exploit we will take advantage so we can 
11    differentiate between different users using the same matching 
12    
13   - Thus, the conclusion is that for managing our state we can use the local storage get/set item. 
14    
15   - Our desired functionality is as follows: 
16       - Create an image tag 
17       - Attach an "onload" event handler to it 
18       - The handler should check for local storage object 
19       (for the specific user), if found any, make the img invisible 
20       (change the style.visibility value to hidden), 
21       else store a local storage object so next time the image will be hidden 
22    
23   - This functionality is performed by sending the following message (on any channel): 
24   <img id="cat_gif" src="https://media.giphy.com/media/l46CaUs3XUKHaPmak/giphy.gif" 
25    onload="if (localStorage.getItem(username) != null) { 
26    document.getElementById('cat_gif').style.visibility='hidden' 
27    } else localStorage.setItem(username, true)" />
